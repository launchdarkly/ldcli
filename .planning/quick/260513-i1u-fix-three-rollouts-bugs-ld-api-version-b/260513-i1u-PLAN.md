# Quick Task 260513-i1u: Phase 1 gap closure (real-server bugfix + smoke)

## Objective

Phase 1 of the milestone (list-foundation-first-end-to-end-slice) shipped Walking Skeleton → Real HTTP → Full UX over three plans, but a user-driven test against real staging revealed three bugs that all unit tests missed:

1. **Missing `LD-API-Version: beta` header.** Server returns 403 because the internal automated-releases API requires it. Confirmed against real staging.
2. **403 swallows the server's `apiBody.Message`.** Generic fallback message hides the real cause (e.g. the beta-header requirement).
3. **Error envelope goes to stderr in JSON mode.** Violates AGENT-04 / D-07 contract — agents must branch on outcome via stdout.

Also: bake a `Real-server validation` constraint into PROJECT.md so future phases don't rely on httptest synthetic servers as the only check.

Phase 1 is already merged on `ae/cli-gr`. This is retroactive gap closure — do NOT update STATE.md / ROADMAP.md / phase plan files.

### Task 3 — Emit error envelope on stdout in JSON mode (AGENT-04)

**Files:** `cmd/flags/rollouts/list.go`, `cmd/flags/rollouts/list_test.go`

**Background:** CONTEXT.md D-07 / AGENT-04 says agents must branch on outcome via stdout. Currently `./ldcli flags rollouts-beta list ... --output json` writes the error envelope to stderr.

**Investigate first:** Find the runE in `list.go`. Trace how a client error becomes an envelope and is written. Likely candidates: a `cmd.PrintErr` / `os.Stderr.Write` call, or a `output.CmdOutputError` invocation that picks stderr. The fix is to route the envelope to stdout when `--output json` is active.

**Action:**
- In runE, when the client returns an error AND output is JSON, marshal an `Envelope` of kind `Error` and write to `cmd.OutOrStdout()` (not `cmd.ErrOrStderr()` / `os.Stderr`). Exit code stays 1 (D-01). Non-JSON output keeps existing stderr behavior (or whatever the plaintext error path does — do NOT change plaintext UX).
- The `--output json` flag suppresses the beta banner already; suppressing the error envelope to stderr is symmetric.
- Add an integration test in `list_test.go` using `cmd.CallCmdWithStderr` (the helper added in Plan 03) that mocks the client to return an error, runs with `--output json`, and asserts: (a) stdout contains the error envelope JSON; (b) stderr does NOT contain the envelope JSON.

**Verify:** `make test` passes. Run the rebuilt binary against a bogus flag with `--output json` — envelope lands on stdout, exit code is 1.

**Commit:** `fix(rollouts): emit error envelope on stdout in JSON mode (AGENT-04)`

**Done:** Test passes; manual `./ldcli flags rollouts-beta list ... --output json 2>/dev/null` shows the envelope.
