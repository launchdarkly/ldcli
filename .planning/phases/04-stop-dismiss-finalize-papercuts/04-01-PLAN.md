## Phase Goal

**As a** human operator or AI agent managing an automated rollout via the LaunchDarkly CLI, **I want to** manually stop the current rollout on a flag to a chosen final variation in one CLI invocation that pre-reads state, refuses to stop a terminal rollout, and returns a confirmation envelope naming the post-mutation state, **so that** I can decisively shepherd a rollout to its conclusion (rolling forward to the target or rolling back to the original) without ever needing the LaunchDarkly UI and without risking a silent no-op against an already-completed rollout.

<objective>
Ship `ldcli flags rollouts-beta stop` end-to-end as a thin vertical slice — command body, typed Client.Stop method (PATCH semantic-patch + re-fetch), flesh out StopInstruction, mock, new error-code constant, new CLI flag, plaintext rendering, JSON envelope with `meta.uiURL`, and full test coverage — so an operator can decisively stop a rollout in one invocation with a structured pre-read refusal for already-terminal rollouts.

Purpose: deliver Phase 4 SC#1, SC#2 (no-rollouts case), SC#3 (already-terminal + no-rollouts agent-friendly errors), and SC#4 (confirmation envelope + meta.uiURL) for the stop verb. This is a vertical-slice plan — one user-visible capability ships end-to-end. Plan 04-02 ships `dismiss-regression` separately because the two semantically distinct verbs touch overlapping internal files (`client.go`, `mock_client.go`, `errors.go`, `instructions.go`, `rollouts.go`), so serializing them is the safer wave assignment.

Output: a working `stop` subcommand reusing the Phase 1 + Phase 2 substrate verbatim (envelope, error mapping, semantic-patch shape, `mapAPIError`), one new typed client method (`Client.Stop`), one new error-code constant (`ErrCodeAlreadyTerminal`), one new CLI flag (`--to-variation`), one new envelope helper (`NewRolloutEnvelopeWithUI` + `BuildUIURL`), and a comprehensive test suite. Sets up Plan 04-02 to add `Client.DismissRegression` against the same substrate.

**Prototype-first framing (carry forward from PROJECT.md):** This is a learnings-vehicle, not a production CLI build. Ship working code fast; capture API papercuts at every workaround site with `// PAPERCUT: PC-NNN` annotations; don't bikeshed architecture. JSON output is API-passthrough — do NOT invent envelopes/classifiers to paper over API gaps in this plan (file new papercuts during Plan 04-03 smoke if any surface).
</objective>
