<objective>
Ship the end-to-end vertical slice for `ldcli flags rollouts-beta start`. The operator (human or agent) composes a `startAutomatedRelease` semantic-patch from CLI flags, the CLI mutates the flag, re-fetches the new rollout, and emits a `rollouts.v1beta1` envelope carrying the new rollout's ID and initial state.

This plan honors:
- D-01 (any error → exit 1; taxonomy via `error.code`)
- D-02 (stages allocation is percent int; CLI multiplies by 1000)
- D-03 (stages duration is Go duration string with mandatory unit suffix)
- D-04 (`--pause-on-regression` / `--revert-on-regression` repeatable; mutex error on overlap)
- D-05 (releaseKind inferred from presence of pause/revert flags; no `--release-kind`)
- D-06 (no `--metric-group` — defer to v1.1)
- D-07 (fallthrough + `--rule-id` only; no `--ref`/`--clauses`)
- D-08 (server-policy "default rule disabled" → `unknown_upstream` — no dedicated code)
- D-09 (NO preflight; NO `recommended-duration` GET; NO `--skip-health-checks`)
- D-10 (NO `Idempotency-Key`; NO `--idempotency-key`; NO `meta.idempotencyKey`)
- D-11 (two-step PATCH + GET-with-filter+limit=1 pattern; robustness specifics implemented)
- D-12 (error-code mapping by server message substring; no client-side pre-fetch of flag state)
- Phase 1 D-07 (JSON-mode errors on stdout, exit 1)
- AGENT-01/AGENT-04 (JSON output parseable; RFC 3339 timestamps in `data.createdAt`)

Purpose: Operator can take a merged feature behind a flag, kick off an automated rollout via the CLI alone, get the new rollout's ID back as parseable JSON, and trust that the CLI's error envelope tells them exactly what failed (and what to do next) when the server rejects the request. This is the milestone's primary feedback loop for the agent-driven workflow.

Output:
- `internal/rollouts/` package exposes a working `Start` method on the Client interface, with hand-edited MockClient and httptest-backed client tests covering the two-step pattern, the empty-refetch retry, and every new error-message mapping.
- `cmd/flags/rollouts/start.go` is a working Cobra subcommand, wired under `flags rollouts-beta`, with table-driven tests for stages parsing, guarded/progressive inference, the mutex-violation usage error, and JSON-mode error-on-stdout (D-07).
- `02-SMOKE.md` captures real-staging evidence (Smoke A–E from RESEARCH §Q10).
</objective>
